## AWS 기반 서버리스 아키텍쳐

~~~~
다음 조건을 만족하는 AWS 기반 서비스를 작성하는 과정을 촬영한 동영상과 람다함수 소스코드(index.js 및 package.json), 사용자 웹 페이지 소스코드(html, js, css 등)를 “OO조-OOO.zip" 파일로 압축해서 제출하시오. (제출 파일명 반드시 준수)

동영상에는 리소스 초기화 결과, 자동으로 변형된 동영상이 저장된 결과, GitHub 로그인 결과(LocalStorage 내용), 동영상 목록 출력 및 실행 결과가 반드시 포함되어 있어야 합니다. 

1. 기존에 생성한 AWS 리소스를 모두 삭제합니다. 
- SNS 주제(topic)와 구독
- API Gateway API
- Lamdba 함수
- S3 버킷에 저장된 객체와 버킷
- Elastic Transcoder 파이프라인

2. 동영상 업로드용 S3 버킷(video-upload)과 변환 파일 저장용 S3 버킷(video-transcoded)을 생성합니다. 

3. video-upload 버킷에 동영상 파일을 추가하면 “System preset: Web” 형식으로 변형된 동영상을 생성해서 video-transcoded 버킷에 저장합니다. 

4. 사용자 웹 페이지를 만들고 auth0.com과 연동하여 SNS 로그인 기능을 구현합니다. 
- SNS 로그인은 GitHub을 포함해 총 3가지 방식을 지원합니다. 

5. 로그인에 성공하면 SNS에 등록된 프로필 사진과 이름, 그리고 video-transcoded 버킷에 저장된 동영상 목록을 출력합니다. (로그인/로그아웃 버튼 토글)

6. 동영상 목록을 클릭하면 해당 동영상을 재생합니다. 

※ 람다 함수 및 사용자 웹 페이지 구현 언어는 제한이 없으며, 편한 언어를 활용하면 됩니다. 
※ 제시한 요구사항 외 모니터링, 로그, 경보 등은 구현하지 않아도 무관합니다.
평가기준
- AWS 리소스를 생성, 삭제할 수 있다. 
- S3 버킷 이벤트를 이용하여 Lambda 함수를 실행할 수 있다.
- Elastic Transcoder의 프리셋을 활용하여 동영상 형식을 변경할 수 있다. 
- auth0.com과 연동하여 소셜 로그인 기능을 구현할 수 있다. 
- Lambda 함수를 이용하여 AWS 리소스를 조작할 수 있다. 
- HTML5 비디오 태그를 사용할 수 있다. 
- jQuery 라이브러리를 이용하여 DOM 스크립팅이 가능하다. 
- 최종 결과물 형식 및 내용이 제시한 요구사항과 동일하다.
